# JSON文件路径问题排查指南

## 🔍 问题现象

运行 `import_all_videos.bat` 时出现：
```
错误: JSON文件或目录不存在: F:\视频json文件\视频json文件\videos_pexels_food.json
```

## ✅ 解决步骤

### 步骤1: 检查文件是否存在

运行路径检查工具：
```cmd
backend-server\scripts\check_json_path.bat
```

这个脚本会检查常见的JSON文件路径，并显示哪些文件存在。

### 步骤2: 确认正确的路径

根据检查结果，确认您的JSON文件实际路径：

**可能的情况**：
1. 文件在F盘，但路径不同
2. 文件在其他盘（如D盘、E盘）
3. 文件名不同
4. 目录结构不同

### 步骤3: 修改批处理脚本

打开 `import_all_videos.bat`，找到这一行：
```bat
set JSON_PATH=F:\视频json文件\视频json文件\videos_pexels_food.json
```

**修改为正确的路径**：

#### 情况1: 文件在其他位置
```bat
set JSON_PATH=D:\我的文件\视频json文件\videos_pexels_food.json
```

#### 情况2: 导入整个目录
```bat
set JSON_PATH=F:\视频json文件\视频json文件
```

#### 情况3: 文件在其他盘
```bat
set JSON_PATH=E:\视频json\videos_pexels_food.json
```

### 步骤4: 使用绝对路径

如果相对路径有问题，使用完整的绝对路径：
```bat
set JSON_PATH=F:\视频json文件\视频json文件\videos_pexels_food.json
```

### 步骤5: 检查中文路径

如果路径包含中文，确保：
1. 批处理文件保存为UTF-8编码（带BOM）
2. 或者使用英文路径

## 🛠️ 手动查找文件路径

### 方法1: 使用文件资源管理器

1. 打开文件资源管理器
2. 导航到JSON文件所在位置
3. 右键点击文件 → 属性
4. 复制"位置"和文件名，组合成完整路径

### 方法2: 使用命令行

```cmd
REM 列出F盘下的所有JSON文件
dir /s /b F:\*.json

REM 或者搜索特定文件名
dir /s /b F:\*videos_pexels*.json
```

### 方法3: 使用PowerShell

```powershell
# 搜索JSON文件
Get-ChildItem -Path F:\ -Filter *.json -Recurse | Select-Object FullName
```

## ⚠️ 注意事项

1. **路径中的反斜杠**: Windows使用 `\`，不是 `/`
2. **路径中的空格**: 如果路径有空格，确保用引号括起来
3. **中文路径**: 确保文件系统支持中文文件名
4. **权限问题**: 确保有读取文件的权限

## 🔧 快速修复

如果文件确实在 `F:\视频json文件\视频json文件\` 目录下，但脚本找不到：

1. **检查文件名大小写**（Windows通常不区分，但最好保持一致）
2. **检查文件扩展名**（确保是 `.json`，不是 `.JSON` 或其他）
3. **尝试使用通配符**：
   ```bat
   set JSON_PATH=F:\视频json文件\视频json文件\*.json
   ```

## 📞 如果还是找不到

1. 运行 `check_json_path.bat` 查看详细检查结果
2. 手动确认文件是否真的存在
3. 尝试将文件复制到项目目录下，使用相对路径：
   ```bat
   set JSON_PATH=backend-server\scripts\videos_pexels_food.json
   ```
